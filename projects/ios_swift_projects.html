The projects listed bellow are following the mentioned training courses content, some of them done on xcode10, some on xcode9. The MacOS used for the purpouse was High Sierra version, launched in Oracle's VirtualBox machine with 8G ram.

Vision-ML is a simple app utilizung the SqeezeNet ML model from CoreML/Vision framework to guess the dominant object on a picture. Speech synthesizer from AVFoundation is added as extention functionality.

Breakpoint-Community-App is a developer styled community app. Firebase back-end ( installed via Cocoa pods ) is used as a storage for building this multiple controller application. Segue transitions were combined with ui view controllers instantiation. Feed posts and user groups were created using table views. Interactive user lists were populated by table view row selections.

Goals-Monitor is a small app allowing the user to pile up goals and monitor easily their execution progress. Basic core data functionality is used to provide the app goals memorization.

Pixel-City allows the user to pin map positions in the close vicinity of his location and get image info what is cashed as photos in that area, so he can visit/ check up at spot later. Map kit functionality and Flickr API were used to build the app, Alamofire and AlamofireImage also used to fasciliate web requests/ images collection. Couple more gizmos like activity/ progress indicators, custom map annotations, pull up views and 3D touch also got in.

Bluetooth-Tracker Simple app based on the abstraction provided by the CoreBluetooth, to monitor/ select available networks around our device.

Snap-Chat Simple clone of the popular app build using the Google's Firebase back-end service. Instant messaging and device image galery access were the accents here.

Uber Clone is also a basic clone of a popular app. Built on top of Google Firebase back-end service, it was a nice exercise in the generation of requests and providing services pattern. MapKit knowledge got extended with the navigate feature.

Tinder Clone is another basic clone of a popular app. Built on top of Parse server hosted on Heroku as back-end service, it was an exercise in the instant messaging, swiping gestures and bits of animation tricks.

Instagram is a basic clone of the popular app. Built on top of Parse server hosted on AWS as back-end service, it was a meet up with Parse Dashboard, getting used to it's basic functionalities - auth services, data storage, PFQueries etc. Images/ camera manipulation were implemented.

Smack-Chat is a live chat application. It came pretty complex compared to my present standards. Prebuilt back-end APIs were hosted on Heroku and tuned to use Mongo DB. The local hosting option also showed up. Those pods came in handy: Alamofire, SwiftyJSON, Socket.IO-Swift-Client. Apart from sweets like SWReveal and db management, the main benefit from this project was the interraction with the APIs and it's debug process assisted by platforms like Postman and RoboMongo.

Eggs-Timer is a simple countdown timer utilizing the timer object, nav/ tab bars and customizing controls.

Window-Shopper is a simple single UI view calculator, if we want to know how long should one work to buy an item he likes, provided his hourly working wage is known in advance. Here IBDesignable view was introduced, Unit testing built in xcode, customizing text fields with nesting items into them.

Dev Profile App is the basic UI layout of a developer console. Single UI view controller with labels and buttons. The take away from this one was the usage of auto layout to allow custom disposition of the UI items depending on the device type/ screen sizes.

Swoosh App is the basic UI layout of a community basketball fans app. Few UI view controllers with labels, buttons and their segues.
